# 2026-04-01 开发日志

## 今日实现

### 1. `/buddy` 改为 `local-jsx` 并增加档案卡 UI

**背景/问题：** `/buddy` 原先为 `type: 'local'`，只能返回纯文本；用户期望与官方类似的「Buddy Profile」卡片（边框、精灵 ASCII、稀有度、属性条等）。

**涉及文件：**
- `src/commands/buddy/index.ts` — 命令类型改为 `local-jsx`、`immediate: true`，`load` 指向 `buddy-ui.js`。
- `src/commands/buddy/buddy-ui.tsx` — 实现 `LocalJSXCommandCall`：子命令分支里对仅需文案的场景 `onDone(..., { display: 'system' })`；展示档案时返回 `<BuddyProfileCard />`。
- `src/buddy/BuddyProfileCard.tsx` — 新建 Ink 卡片：`renderSprite` 第 0 帧、`RARITY_STARS`/`STAT_NAMES` 条形图、`useInput` 监听 Esc/Enter 关闭。
- `src/commands/buddy/buddy.ts` — 已删除，逻辑迁入 `buddy-ui.tsx`。

**技术选型：**

| 方案 | 优点 | 缺点 |
|------|------|------|
| 继续在 `local` 里返回长文本 | 改动面最小 | 无法达到卡片级排版与交互 |
| **`local-jsx` + 独立卡片组件** | 与 `/status`、`/btw` 一致；可复用 `renderSprite`、主题色 | 需拆分 `call` 签名与 `onDone` 约定；`feature('BUDDY')` 仍须满足 Bun 的 ternary 用法 |

选型理由：富 UI 只能走 `local-jsx`；`immediate: true` 避免排队，与现有即时命令体验一致。

**实现要点：** `showCard` 在 `saveGlobalConfig` 后调用 `getCompanion()` 再渲染，保证与磁盘一致；`pick`/`auto`/孵化成功路径统一走卡片；`list`/`help`/校验失败走 `display: 'system'`。

**实现要点：** 技能正文要求「先 docs 再 developerWords」；执行时以终端日期为准，避免模型臆测当天。

## 关键收获

1. **`local` vs `local-jsx`：** 终端富 UI 必须注册为 `local-jsx`，`call(onDone, context, args)` 返回 `ReactNode`，纯文本用 `onDone` 的 `display: 'system' | 'skip'` 控制是否进 transcript。
2. **Bun `feature()`：** 只能出现在 `if`/三元中；卡片文案里提示 BUDDY 开关时保持内联三元，避免中间变量赋值触发打包器报错。
3. **每日记录技能：** 日期一律 `date '+%Y-%m-%d'`；发言文件严格只收录用户原话，长段粘贴可与仓库内 SKILL 对齐以保证「原文」一致性。
